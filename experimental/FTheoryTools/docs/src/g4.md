# ``G_4``-Fluxes

In F-theory compactifications on Calabi–Yau 4-folds, **``G_4``-fluxes** are essential ingredients for defining consistent and physically meaningful vacua. They affect various physical aspects, such as the chiral spectrum in the 4D effective theory, tadpole cancellation, and moduli stabilization. This page provides an accessible introduction to working with ``G_4``-fluxes in `FTheoryTools`, including background, construction methods, and available functionality.

---

## Background: What Is a ``G_4``-Flux?

Let ``\widehat{Y}_4`` be a smooth Calabi–Yau 4-fold obtained via crepant resolution of a singular elliptically fibered 4-fold ``Y_4 \to B_3``. A **``G_4``-flux** is a cohomology class:

$G_4 \in H^4(\widehat{Y}_4, \mathbb{Q}) \cap H^{2,2}(\widehat{Y}_4)$

subject to certain **quantization** and **transversality** conditions:

1. **Quantization (Witten's condition):** ``G_4 + \frac{1}{2} c_2(\widehat{Y}_4) \in H^4(\widehat{Y}_4, \mathbb{Z})``

2. **Transversality:** The flux must be orthogonal to both the base and the fiber in a precise cohomological sense.

3. **Gauge Symmetry Compatibility:** In model building, one often demands that the ``G_4``-flux preserves the non-abelian gauge symmetry encoded in the fibration.

These requirements ensure that the flux is consistent with physical constraints such as anomaly cancellation and the D3-brane charge quantization. For further theory and background, see the exposition in [Weigand 2018](@cite Wei18).

## Constructing ``G_4``-Fluxes

In `FTheoryTools`, ``G_4``-fluxes are built from cohomology classes and associated to a given resolved model. You can construct a flux directly via:

```@docs
g4_flux(model::AbstractFTheoryModel, class::CohomologyClass)
```

For models derived from the **quadrillion F-theory Standard models (QSMs)** (cf. [Cvetič, Halverson, Ling, Liu, Tian 2019](@cite CHLLT19)), a shortcut constructor is:
```@docs
qsm_flux(qsm_model::AbstractFTheoryModel)
```

## Exploring ``G_4``-Flux Data

Once a `G4Flux` object is created, the following accessors are available:

```@docs
model(gf::G4Flux)
cohomology_class(gf::G4Flux)
d3_tadpole_constraint(gf::G4Flux; check::Bool = true)
g4_flux_family(gf::G4Flux; check::Bool = true)
```

The family associated to a flux expresses it in terms of an integral-rational linear combination over a basis. You can extract these coefficients via:

```@docs
integral_coefficients(gf::G4Flux)
rational_coefficients(gf::G4Flux)
```

## Working with Families of ``G_4``-Fluxes

A common use case is the analysis of entire families of fluxes. These are parameterized by integral and rational coefficient matrices:

### Constructor

```@docs
family_of_g4_fluxes(m::AbstractFTheoryModel, mat_int::QQMatrix, mat_rat::QQMatrix, offset::Vector{QQFieldElem}; check::Bool = true)
```

### Attributes

```@docs
model(gf::FamilyOfG4Fluxes)
matrix_integral(gf::FamilyOfG4Fluxes)
matrix_rational(gf::FamilyOfG4Fluxes)
d3_tadpole_constraint(fgs::FamilyOfG4Fluxes; check::Bool = true)
```

### Properties

```@docs
is_well_quantized(fgs::FamilyOfG4Fluxes; check::Bool = true)
passes_transversality_checks(fgs::FamilyOfG4Fluxes; check::Bool = true)
breaks_non_abelian_gauge_group(fgs::FamilyOfG4Fluxes; check::Bool = true)
```

You can generate individual flux instances within a family of ``G_4`` fluxes as follows:

```@docs
flux_instance(fgs::FamilyOfG4Fluxes, int_combination::ZZMatrix, rat_combination::QQMatrix; check::Bool = true)
random_flux_instance(fgs::FamilyOfG4Fluxes; check::Bool = true)
```

To construct a physically viable flux randomly:

```@docs
random_flux(m::AbstractFTheoryModel; not_breaking::Bool = false, check::Bool = true)
```

## Basis Construction in the Ambient Space

In practice, ``G_4``-fluxes are often constructed from ambient space cohomology:

- Let ``X_\Sigma`` be a smooth, complete, simplicial toric variety
- Let ``\widehat{Y}_4 = \mathbb{V}(p) \subset X_\Sigma``

We define candidate cycles as intersections ``A = \mathbb{V}(x_i, x_j)``. Their restriction to ``\widehat{Y}_4`` is tested for triviality via:

```@docs
chosen_g4_flux_basis(m::AbstractFTheoryModel; check::Bool = true)
```

Supporting methods:

```@docs
basis_of_h22_ambient(m::AbstractFTheoryModel; check::Bool = true)
basis_of_h22_hypersurface(m::AbstractFTheoryModel; check::Bool = true)
basis_of_h22_ambient_indices(m::AbstractFTheoryModel; check::Bool = true)
basis_of_h22_hypersurface_indices(m::AbstractFTheoryModel; check::Bool = true)
converter_dict_h22_ambient(m::AbstractFTheoryModel; check::Bool = true)
converter_dict_h22_hypersurface(m::AbstractFTheoryModel; check::Bool = true)
```

### Note on Rational Equivalence

Naively intersecting ``\mathbb{V}(p)`` and ``\mathbb{V}(x_i, x_j)`` might lead to subtle artifacts if the cycles are not transverse. These cases are handled using linear relations in ``X_\Sigma`` to replace cycles with rational equivalents to restore transversality.

## Special Flux Families

To automatically generate flux families satisfying all physical conditions:

```@docs
special_flux_family(m::AbstractFTheoryModel; not_breaking::Bool = false, check::Bool = true)
```

This method filters fluxes that are:

- Well-quantized
- Satisfy transversality
- (Optionally) Preserve the non-abelian gauge group

!!! warning
    This operation can be computationally intensive on large geometries.
